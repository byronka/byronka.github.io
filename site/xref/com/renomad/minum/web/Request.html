3   /**
4    * An HTTP request.
5    * <p>
6    * From <a href="https://en.wikipedia.org/wiki/HTTP#Request_syntax">Wikipedia</a>
7    * </p>
8    *<p>
9    *     A client sends request messages to the server, which consist of:
10   *</p>
11   * <ul>
12   *     <li>
13   *      a request line, consisting of the case-sensitive request
14   *      method, a space, the requested URL, another space, the
15   *      protocol version, a carriage return, and a line feed, e.g.:
16   *      <pre>
17   *          GET /images/logo.png HTTP/1.1
18   *      </pre>
19   *      </li>
20   *
21   *      <li>
22   *      zero or more request header fields (at least 1 or more
23   *      headers in case of HTTP/1.1), each consisting of the case-insensitive
24   *      field name, a colon, optional leading whitespace, the field
25   *      value, an optional trailing whitespace and ending with a
26   *      carriage return and a line feed, e.g.:
27   *
28   *      <pre>
29   *      Host: www.example.com
30   *      Accept-Language: en
31   *      </pre>
32   *      </li>
33   *
34   *      <li>
35   *      an empty line, consisting of a carriage return and a line feed;
36   *      </li>
37   *
38   *      <li>
39   *      an optional message body.
40   *      </li>
41   *      In the HTTP/1.1 protocol, all header fields except Host: hostname are optional.
42   * </ul>
43   *
44   * <p>
45   * A request line containing only the path name is accepted by servers to
46   * maintain compatibility with HTTP clients before the HTTP/1.0 specification in RFC 1945.
47   *</p>
48   */
49  public record Request(Headers headers, RequestLine requestLine, Body body,
50                        /*
51                        This is the remote address making the request
52                         */
53                        String remoteRequester) {
